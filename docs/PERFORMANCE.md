# Scan Performance Notes (v0.7.2)

Every number below was measured with the reproducible benchmark included in this
repository. No performance claim is made beyond this data.

## Measured results

Environment: Windows 10 Pro, Python 3.13, NTFS, local disk. Synthetic workspaces
generated by `repo-cleanroom demo-workspace` (small repos: a handful of files per
artifact; sizes below are dominated by per-directory walk overhead, not file I/O).

| Synthetic repos | Artifacts found | `scan` time | `plan` time |
|---:|---:|---:|---:|
| 100 | 396 | 0.74 s | 0.47 s |
| 500 | 1,996 | 3.62 s | 2.37 s |

Observed scaling in this range is roughly linear in repository count for both
commands.

## Known limitations of these numbers

- The synthetic repositories are intentionally tiny. A real `node_modules` or
  `target` directory can contain tens of thousands of files; on such workspaces
  the size-estimation walk (which visits every file without following symlinks)
  will dominate, and per-repo time will be much higher than in this table.
- Results depend on disk type, antivirus interception, and NTFS state. Treat the
  table as an order-of-magnitude reference, not a guarantee.
- No memory profiling has been performed yet.

## Reproduce

```powershell
py scripts/benchmark_scan.py --repo-count 100 --work-dir C:\path\to\fresh\dir
```

The script writes only inside the working directory you name (which must be
fresh), generates only synthetic data, and deletes nothing.
